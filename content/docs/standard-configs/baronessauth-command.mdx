# Файл `baronessauth.yml`

**Путь:** `content/BaronessAuth/config/commands/baronessauth.yml`

Команда `/baronessauth` (`/auth`). Право: `BaronessAuth.command.baronessauth`. Предназначена для админов.

Так как команд очень много, детальную настройку каждой субкоманды убрали - чтобы конфиг не был на тысячу строк.

## 🔧 Основные секции

### Help Header/Footer
Заголовки справки:
- **help-header** - заголовок списка команд
- **help-footer** - подвал списка команд

### Reload
Команда перезагрузки:
- **help** - справка по команде
- **success** - сообщение об успешной перезагрузке

### Player Info
Информация об игроке:
- **help** - справка по команде
- **output** - формат вывода информации
- **licensed-yes/no** - отображение статуса лицензии

### Player Setpassword
Установка пароля игроку:
- **help** - справка по команде
- **success** - сообщение об успешной смене пароля

### Player Delete
Удаление регистрации игрока:
- **help** - справка по команде
- **kick-reason** - причина кика при удалении
- **success** - сообщение об успешном удалении

### Backup Info
Информация о бекапе:
- **help** - справка по команде
- **output** - формат вывода информации

### Backup Create
Создание бекапа:
- **help** - справка по команде
- **progress** - отображение прогресса
- **success** - сообщение об успешном создании

## 🔐 Доступные команды

### Основные команды
- `/auth reload` - перезагрузить плагин
- `/auth player <игрок> info` - информация об игроке
- `/auth player <игрок> setpassword <пароль>` - установить пароль
- `/auth player <игрок> delete` - удалить регистрацию

### Бекапы
- `/auth backup info <файл>` - информация о бекапе
- `/auth backup create` - создать бекап
- `/auth backup load <файл>` - загрузить бекап

## ⚠️ Важно

- Все команды требуют соответствующих прав
- Команда `reload` не перезагружает настройки базы данных
- Удаление регистрации кикает игрока с сервера
- Бекапы создаются асинхронно с отображением прогресса
